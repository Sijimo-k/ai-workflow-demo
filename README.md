# EC受注処理 自動化パイプライン（order-pipeline）

複数ECモール（**楽天市場 / Yahoo!ショッピング / Shopify**）の受注CSVを取り込み、
**統合 → 発送対象の抽出 → 出荷リスト作成 → 顧客向けメール文面生成**までを
ワンコマンドで自動化するデモです。

> Claude Code を使った副業案件応募用ポートフォリオの作品です。
> 「何を作ったか」よりも **「どんな業務課題を、どの工程をAIで進めて解決したか」** を示すことを主眼にしています。

---

## 解決する業務課題

| Before（手作業） | After（本ツール） |
| --- | --- |
| モールごとに管理画面を開いてCSVをダウンロード | 3モールのCSVをまとめて1コマンドで処理 |
| 列名・文字コードがバラバラで目視転記、ミスが発生 | 統一スキーマへ自動正規化（転記ゼロ） |
| 「入金済みか」「発送済みでないか」を1件ずつ確認 | 発送対象を自動抽出（条件判定の属人化を解消） |
| 顧客への発送通知メールを毎回手書き | テンプレート差し込みで全件自動生成 |
| **1件あたり 約5分** | **1件あたり 約30秒相当（実処理は全件で1秒未満）** |

実務で詰まるのは「3モールで**列名・文字コード・ステータス表記がすべて違う**」点です
（楽天/Yahooは CP932、Shopifyは UTF-8＋英語列名）。この正規化こそが自動化の中心的価値で、
本ツールはモールごとのローダにその差異を閉じ込めています。

---

## 処理フロー

```
楽天CSV(CP932) ┐
Yahoo CSV(CP932)├─► [ローダ:正規化] ─► 統合Order一覧 ─► [発送対象抽出] ─┬─► 出荷リスト(Excel)
Shopify CSV(UTF-8)┘     (列名/文字コード/        (入金済み&未発送)        ├─► 統合CSV
                        ステータスを吸収)                                  └─► 発送通知メール(.txt × 件数)
```

「発送対象」の定義 = **入金済み（PAID）かつ 未発送（UNFULFILLED）**。
未入金や発送済みは自動的に除外されます。

## 使い方

### 1. セットアップ（[uv](https://docs.astral.sh/uv/) 利用）

```sh
uv venv
uv pip install -e ".[dev]"
```

### 2. サンプル受注データの生成

```sh
uv run python scripts/generate_sample_data.py
```

`data/sample/` に3モールぶんのCSV（わざと文字コード・列名を変えてある）が作られます。

### 3. 実行

```sh
uv run python -m order_pipeline \
  --rakuten data/sample/rakuten_orders.csv \
  --yahoo   data/sample/yahoo_orders.csv \
  --shopify data/sample/shopify_orders.csv \
  --outdir  output
```

`output/` に以下が生成されます。

| 生成物 | 内容 |
| --- | --- |
| `unified_orders.csv` | 全モール統合済みの受注一覧（Excelで開いても文字化けしないUTF-8 BOM） |
| `shipping_list.xlsx` | 発送対象だけの出荷リスト（ヘッダ装飾・列幅・合計行つき） |
| `mails/*.txt` | 発送対象ごとの顧客向け発送通知メール文面 |

実行すると、件数・発送対象数・合計金額・**手作業比の削減見込み**がコンソールに表示されます。


## 構成

```
ai-workflow-demo/
├── src/order_pipeline/
│   ├── models.py      # 統合受注スキーマ（Order / 各種 Enum）
│   ├── loaders.py     # モール別CSV → Order 正規化（文字コード・列名差を吸収）
│   ├── pipeline.py    # 統合・発送対象抽出・集計（純関数）
│   ├── exporters.py   # CSV / Excel / メール文面の出力
│   ├── mail.py        # 発送通知メールのテンプレート生成
│   └── cli.py         # コマンドライン実行
├── scripts/
│   └── generate_sample_data.py  # デモ用サンプルCSV生成
├── tests/             # pytest（正規化・抽出・メール）
└── data/sample/       # サンプル受注CSV（生成物）
```

## 設計上の判断

- **メール生成はテンプレート方式**（LLM不使用）。API キー不要で誰でも再現実行でき、文面のブレや
  転記ミスが構造的に起こらない。「定型業務の自動化」という主旨に最も合うため。
- **統一スキーマ（`Order`）に正規化**し、以降の処理はモールを意識しない。新モール追加は
  `loaders.py` にローダを1つ足すだけで済む（影響範囲をローダ層に限定）。
- **コア処理は副作用のない純関数**にして、ファイル入出力（`exporters.py`）と分離。テストを容易にした。
